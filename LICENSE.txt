TERMS AND CONDITIONS FOR EMOTIONALLY SAFE SOFTWARE DEVELOPMENT

This software is additionally licensed under the Palimpsest License v0.8,
which provides enhanced protections for contributors' emotional safety and
project governance transparency.

Key Provisions:

1. REVERSIBILITY REQUIREMENT
   - All contributions must be reversible within 90 days
   - Contributors retain rights to withdraw their work
   - Maintainers must provide clear reversion procedures

2. EMOTIONAL TEMPERATURE DISCLOSURE
   - Project must maintain public "emotional temperature" metrics
   - Metrics include: contributor churn rate, issue resolution time,
     code review turnaround, governance decision transparency
   - See: MAINTAINERS.md for current metrics

3. CONTRIBUTION PERIMETER FRAMEWORK (TPCF)
   - This project operates at Perimeter 3: Community Sandbox
   - Open contribution with graduated trust model
   - See: CONTRIBUTING.md for detailed access levels

4. NO COERCED LABOR
   - All contributions must be voluntary
   - No implicit or explicit pressure on contributors
   - Maintainers must respect contributor boundaries

5. TRANSPARENT GOVERNANCE
   - All governance decisions documented publicly
   - Community input required for major changes
   - See: MAINTAINERS.md for decision-making process

6. ATTRIBUTION PRESERVATION
   - All contributors properly credited in .well-known/humans.txt
   - Git history preserved unless contributor requests removal
   - Derivative works must maintain attribution chain

7. ANTI-HARASSMENT COMMITMENT
   - Zero tolerance for harassment, discrimination, or abuse
   - See: CODE_OF_CONDUCT.md for enforcement procedures
   - Safe reporting channels with confidentiality
